# BMO - 아이 버전 기획

## 프로젝트 개요

어린 자녀가 있는 가정을 대상으로 한 BMO의 파생 버전.
아이의 호기심을 해소해주는 AI 친구이자, 부모와 아이 사이의 소통 다리 역할을 수행한다.

**타겟**: 유아 (3~5세)

---

## 핵심 가치 제안

> "아이의 호기심을 채워주고, 지친 부모를 쉬게 해주는 가족 AI"

| 대상 | 가치 |
|------|------|
| 아이 | 호기심을 즉시 해소해주는 친구 |
| 부모 | 아이가 오늘 어떤 생각을 했는지 알 수 있는 창구 + 육아 부담 경감 |
| 관계 | 부모-자녀 대화 소재 제공 |

---

## 기능 목록

### 1. 물체 인식 + 아이 눈높이 설명
아이가 물체를 카메라에 보여주면 BMO가 인식하고 나이에 맞는 언어로 설명해준다.

- **입력**: 카메라에 물체/그림/장난감 등을 들이밈
- **처리**: 멀티모달 LLM (Vision API) → 아이 눈높이 프롬프팅
- **출력**: 음성 설명 + BMO 표정 변화
- **특징**
  - 아이가 이름을 모르는 것도 설명 가능 (말 대신 보여주기)
  - 의성어/의태어 포함, 아주 짧고 단순한 문장으로 설명
  - 같은 물체를 반복해서 보여줄 경우 매번 다른 방식으로 설명 (지루함 방지)
  - 답변 출력 전 연령 적절성 자동 검토 필터링 적용

### 2. 웨이크워드 호출
아이가 BMO를 부르면 대화를 시작한다. 유아의 발음 미숙 및 부정확한 발화를 고려해 설계한다.

- **설계 방향**: 퍼지 매칭 기반 웨이크워드 인식 (정확한 발음이 아니어도 인식)
- **예시**: "비모", "뷔모", "BMO야" 등 유사 발화 모두 인식 대상

### 3. 부모 목소리로 동화 읽어주기
부모의 목소리를 학습해 동화를 부모 목소리로 읽어준다.

- **설정**: 부모가 3~5분 분량의 음성을 녹음 → 1회 등록
- **처리**: ElevenLabs Voice Cloning API → 부모 목소리 TTS 생성
- **출력**: 부모 목소리로 동화 재생
- **가치**: 부모가 자리를 비우거나 피로한 날에도 아이가 익숙한 목소리로 잠들 수 있음

### 4. 오늘의 BMO 리포트 (부모 알림)
하루 동안 아이와 나눈 대화와 탐구 내용을 저녁에 부모에게 요약해서 전달한다.

- **입력**: 하루 동안 쌓인 대화/인식 로그
- **처리**: LLM 요약 → 감성적 문장으로 가공
- **출력**: 카카오 알림톡 or 앱 푸시
- **전달 시점**: 저녁 일정 시간 (예: 오후 6시) 또는 부모 귀가 감지 시
- **리포트 예시**
  ```
  [오늘의 BMO 리포트 - 3월 17일]

  🦕 오늘 가장 많이 궁금해한 것: 공룡
     → 티라노사우루스, 브라키오사우루스 2번 보여줬어요

  🌤️ 이런 질문도 했어요
     → "하늘은 왜 파래요?"
     → "개미는 왜 줄 서서 가요?"

  💬 오늘의 한마디
     → "BMO, 공룡이 진짜 있으면 좋겠다고 했어요"
  ```
- **장기 활용**: 리포트가 쌓이면 아이 성장 기록 앨범으로 활용 가능

## 구현 난이도 분석

> 기준: Python 기초 수준, 실질 2인 개발, 개발 기간 2개월

| 기능 | 난이도 | 주요 도전 과제 |
|------|--------|--------------|
| 물체 인식 + 설명 | ★★★☆☆ | RPi5 카메라 연동, 촬영 트리거 UX 설계, 프롬프트 엔지니어링 |
| 웨이크워드 호출 | ★★★★☆ | 유아 발음 개인차가 크고 예측 불가 → 오탐/미탐 조율이 어려움 |
| 부모 목소리 동화 | ★★☆☆☆ | ElevenLabs API 연동 자체는 쉬움, 녹음 온보딩 흐름 설계 필요 |
| 오늘의 리포트 | ★★☆☆☆ | 대화 로그 저장 + LLM 요약 + 카카오 API, 모두 단순한 구조 |

### 주요 리스크

**웨이크워드** — 가장 큰 기술적 리스크

유아 발음은 개인마다 편차가 크고 같은 아이도 매번 다르게 발음함. 퍼지 매칭 임계값 조정이 까다롭고, 오탐(부르지 않았는데 반응)과 미탐(불렀는데 무반응) 사이에서 균형 잡기 어려움.

→ 대안: STT로 변환 후 Levenshtein 거리 기반 유사도 비교 방식으로 구현. 완벽하지 않아도 데모 수준에서는 허용 가능.

**물체 인식 트리거** — UX 설계 필요

아이가 "지금 보여줄게"를 BMO에게 어떻게 알리는가? 버튼 누르기 / 특정 말 하기 / 항상 켜진 카메라 중 선택이 필요하며, 유아가 직관적으로 쓸 수 있어야 함.

## 개발 일정 (2개월 기준)

| 주차 | 내용 | 담당 |
|------|------|------|
| 1주차 | 환경 구축: RPi5 카메라 모듈 연동, STT/TTS 기본 파이프라인 | 전체 |
| 2~3주차 | **물체 인식 + 설명** 구현 (핵심 기능 우선) | SW |
| 4주차 | **웨이크워드** 구현 및 유사 발음 튜닝 | SW |
| 5주차 | **부모 목소리 동화** (ElevenLabs 연동 + 온보딩 흐름) | SW |
| 6주차 | **오늘의 리포트** (로그 저장 + LLM 요약 + 카카오 API) | SW |
| 7주차 | 통합 테스트 + 버그 수정 + 완성도 보완 | 전체 |
| 8주차 | 발표 준비 + 데모 시나리오 최종 점검 | 전체 |

### 우선순위 원칙

발표 당일 정상 작동이 최우선. 일정이 밀릴 경우 아래 순서로 기능을 축소:

```
1순위 (반드시): 물체 인식 + 설명
2순위 (반드시): 오늘의 리포트
3순위 (여유 시): 부모 목소리 동화
4순위 (여유 시): 웨이크워드 퍼지 매칭 (기본 STT 방식으로 대체 가능)
```

## 기술 스택 (예상)

| 구성요소 | 기술 |
|---------|------|
| 물체 인식 | 멀티모달 LLM Vision API (GPT-4o / Gemini Vision) |
| 웨이크워드 | 퍼지 매칭 기반 음성 인식 |
| 음성 입력 | STT (Whisper 등) |
| 응답 생성 | LLM + 아이 눈높이 프롬프팅 + 안전 필터링 |
| 음성 출력 | TTS |
| 부모 목소리 복제 | ElevenLabs Voice Cloning API |
| 부모 알림 | 카카오 알림톡 API |
| 디스플레이 | BMO 표정 UI |
| 하드웨어 | Raspberry Pi 5 + 카메라 모듈 |
