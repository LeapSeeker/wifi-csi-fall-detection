# B-MO 프로젝트 기획안

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 제품명 | B-MO (비모) |
| 모티브 | 핀과 제이크의 어드벤처 타임 — BMO 캐릭터 외관 + 자비스 기능 |
| 핵심 컨셉 | 말하지 않아도 사용자의 상태를 감지하고, 먼저 케어하는 AI 동반자 |
| 포지셔닝 | Passive Ambient Intelligence — 명령 기반이 아닌 상태 감지 기반 |
| 타겟 | 재택근무자 |
| 목표 | 졸업작품 완성 + 테크페어 대회 출품 |

> **핵심 한 줄**: "당신이 괜찮다고 말하기 전에, B-MO는 이미 알고 있다"
>
> 기존 AI 스피커(Alexa, Google Nest 등)는 사용자가 말해야 동작한다.
> B-MO는 카메라(자세) + 마이크(목소리 상태) + WiFi CSI(재실/움직임)를 융합해
> 사용자가 명령하지 않아도 번아웃 징후를 감지하고 먼저 개입한다.

## 3. 하드웨어 구성

| 장비 | 역할 |
|------|------|
| 라즈베리파이5 | 메인 컴퓨팅 (온디바이스 추론) |
| 카메라 모듈 | 자세 감지, 착석 시간, 외출 모드 녹화 |
| 마이크 | 음성 입력 (STT) + 목소리 상태 분류 |
| 스피커 | 음성 출력 (TTS) |
| ESP32-S3 | WiFi CSI 데이터 수집 (재실/움직임 감지) |

- BMO 외관 제작 예정 (어드벤처 타임 캐릭터 참고)
- 추가 장비는 학교 지원 + 개인 사비 50만원 이내

## 4. 기술 스택

| 분류 | 기술 | 비고 |
|------|------|------|
| 언어 | Python | |
| 서버 | Flask + MySQL | 사용자 관리, 일정 DB, 알림 API |
| 웹 프론트엔드 | Flask Jinja2 + HTML/CSS/JS | 일정 관리, 설정, 카메라 영상 확인 |
| 음성 인식 (STT) | Whisper | 로컬 실행 (RPi5) |
| 집중 상태 분류 | 파인튜닝 모델 (음성 기반) | 핵심 AI — 데이터셋/모델 선정 필요 |
| 응답 생성 | LLM API 연동 | 클라우드 API 방식 |
| 음성 출력 (TTS) | 미정 | 별도 논의 필요 |
| 영상 처리 | OpenCV + MediaPipe | 자세 감지, 착석 시간 |
| WiFi 센싱 | ESP32-CSI-Tool | 재실/움직임 감지 (카메라 사각지대 보완) |
| 알림 전송 | 카카오 알림톡 | 앱 없이 카카오톡으로 푸시 수신 |
| 날씨 API | 기상청 API 또는 OpenWeatherMap | |

## 5. 서비스 구조

```
[센서 레이어]
├── 마이크 → STT + 집중 상태 분류 (파인튜닝)
├── 카메라 → 자세 감지 / 착석 시간 / 외출 모드 녹화
└── ESP32-S3 → WiFi CSI → 재실 / 움직임 감지

        ↓ 멀티모달 융합 (RPi5 온디바이스)

[상태 추론 엔진]
└── 집중도 + 자세 + 재실 데이터 종합 → 번아웃 징후 판단
        │
        ├── 정상: BMO 표정 유지, 환경 자동 조절 (조명/음악)
        └── 이상: 휴식/작업전환 알림 + LLM 맞춤 멘트 생성

[Flask 서버 + MySQL]
├── [웹 UI] ← 일정 관리 / 설정 / 카메라 영상 확인
│    └── 모바일 브라우저에서도 접속 가능
└── [카카오 알림톡] ← 이상 감지 / 리마인더 푸시 알림
```

## 6. 기능 목록 (재택근무자용)

### 핵심 기능 (컨셉 직결 — 반드시 구현)

| 기능 | 이유 |
|------|------|
| 멀티모달 상태 감지 (음성+카메라+WiFi CSI 융합) | Passive AI의 핵심 — 명령 없이 상태 파악 |
| 집중 상태 분류 파인튜닝 (음성 기반) | AI가 사용자를 이해하는 핵심 |
| 자세 + 착석 시간 감지 | 신체 컨디션 케어 |
| 자세 교정 알림 + 스트레칭 추천 | 허리 건강 케어 — 자세 불량 지속 시 교정 유도 및 맞춤 스트레칭 제안 |
| 통합 휴식 / 작업전환 알림 | 웰빙 핵심 출력 |
| 조명 무드 제어 | 환경 케어 (JARVIS 요소) |
| 음악 재생 | 환경 케어 (JARVIS 요소) |
| BMO 디스플레이 + 표정 | 동반자 감성 (BMO 요소) |
| 음성 명령 (STT + LLM + TTS) | 모든 기능의 인터페이스 |

### 지원 기능 (웰빙과 연결되므로 유지)

| 기능 | 연결 이유 |
|------|-----------|
| 일정 관리 (음성 + 웹) | 마감 스트레스 예방 → 웰빙 연결 |
| 작업 전환 추천 (상태 연동형, TODO 기반) | 번아웃 감지 시 현재 상태에 맞는 가벼운 작업으로 LLM이 전환 추천 |
| 날씨 정보 | 외출 준비 → 가벼운 케어 |

### 보안 기능 (팀 결정으로 유지)

- **재택 모드 (기본)**: 보안 기능 비활성
- **외출 모드**: 음성 명령으로 전환
  - 카메라: BMO 주변 공간 움직임 감지 + 자동 녹화
  - WiFi CSI: 카메라 사각지대 침입자 유무 감지
  - 이상 감지 시 사용자에게 알림 전송

## 7. 개발 우선순위 (2개월 기준)

> 집중 상태 분류 파인튜닝은 학습 시간이 길기 때문에 전체 기간에 걸쳐 병행 진행

```
1단계 (1~2주) - 기반 세팅
├── 라즈베리파이5 환경 구성 및 라이브러리 설치
├── Whisper STT 기본 동작 확인
├── TTS 기본 동작 확인
├── Flask 서버 + MySQL 기본 구조 구성
└── 카메라 모듈 기본 동작 확인

2단계 (2~5주) - 핵심 AI + 핵심 기능 [병행]
├── [AI 담당] 집중 상태 분류 데이터셋 선정 → 파인튜닝 → 성능 검증 (전 기간 병행)
├── [기능 담당] LLM API 연동 + 응답 흐름 구성
├── [기능 담당] 일정 관리 CRUD + 리마인더 (DB 연동)
├── [기능 담당] 날씨 API 연동
└── [기능 담당] 음성 메모 / TODO 등록

3단계 (4~6주) - 상태 감지 융합 + 보안 + 웹 UI
├── 자세 감지 (OpenCV + MediaPipe)
├── 자세 불량 감지 → 교정 알림 → 스트레칭 추천 로직
├── 착석 시간 감지 로직
├── 멀티모달 융합 → 번아웃 판단 로직
├── 상태 연동형 작업 전환 추천 (집중도 + 자세 + 착석 시간 → LLM 추천)
├── 통합 알림 로직 (종합 판단 → 맞춤 멘트)
├── 조명 / 음악 연동
├── 카메라 움직임 감지 + 외출 모드 전환 로직
├── 카카오 알림톡 연동
└── 웹 UI 구현 (일정 관리 / 설정 / 카메라 영상 확인)

4단계 (6~7주) - WiFi CSI 센싱
├── ESP32-S3 펌웨어 세팅 + CSI 데이터 수집
├── 재실/움직임 감지 로직 → 멀티모달 융합 연동
└── 외출 모드 침입자 감지 연동

5단계 (7~8주) - 통합 마무리
├── 전체 시스템 통합 테스트
├── 버그 수정 및 데모 안정화
└── BMO 외관 제작
```

## 8. 대회 차별화 포인트

- **Passive Ambient Intelligence**: 명령 없이 상태를 감지 — 기존 AI 스피커와 근본적 차이
- **멀티모달 융합 (음성+카메라+WiFi CSI)**: 단일 센서보다 정확한 상태 추론, 학부 수준에서 희귀한 시도
- **WiFi CSI (ESP32-S3)**: 카메라 없이 벽 너머 재실/움직임 감지 — 프라이버시 보호 + 기술적 차별화
- **온디바이스 추론 (RPi5)**: 민감한 데이터 클라우드 미전송 → 프라이버시 강점
- **BMO 캐릭터 외관**: 발표 현장 임팩트 + 동반자형 디바이스 스토리텔링
